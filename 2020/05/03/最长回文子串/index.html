Manacher算法的时间复杂度为 O(n)

文本预处理

如bob —> $#b#o#b，noon -> $#n#o#o#n#,这样无论是奇数回文还是偶数回文，处理之后得到的字符串长度是奇数。这里的插空符号是任意的。我在这里使用#号。$符号是为了找到原始字符串中回文串的起始位置。

理解回文半径

假设字符串s,经过以上的文本预处理之后的字符串t。在使用Manacher算法时，需要开辟一个容量为字符串t长度的数组p来保存回文半径。什么是回文半径？例如p[i]=1，则表示以t[i]为中心的回文半径是1，即回文子串就是t[i]本身。这里有’两’个”完美的规律”：

  • t[i]的回文半径减1等于回文串的长度
  • s回文串的起始位置是(i-p[i])/2.

算法核心

对于处理之后的字符串t,求解回文半径数组p需要两个辅助变量，分别是index和maxlen。index用来表示当前最长回文子串的中心点在t中的下标，maxlen表示当前最长的回文子串右端在t中的下标。在求解过程中遍历字符串的每一个字符t[i]

  • 对于 i < maxlen，即当前最长回文子串覆盖了i，根据回文子串的对称性p[i]=min(2*index-i,maxlen-i)。因为这个时候我们只能确保p[i]的至多等于maxlen-i，至于maxlen后面的是不是回文，那么只能乖乖匹配了。

  • 对于i>=maxlen的位置，只能初始化p[i]=1,然后乖乖去匹配。
